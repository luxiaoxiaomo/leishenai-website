常用技能分享

把 AI 能力做成可复用的工作方法

这里收录我在 AI x HR、知识管理、自动化和个人网站建设中经常使用的技能。每个技能都给出适用场景和上手步骤，后续可以逐个扩展成教程。

任务执行AI Agent + 自动化完成任务

技能卡片

先放底层能力，再放工具流程，最后放可对外交付的发布能力。

01 知识管理

Obsidian 个人知识库

把课程、报告、Prompt 和项目笔记放进同一个知识网络，方便后续写作、备课和拆解案例。

适合场景适合课程研发、报告整理、长期写作和个人知识沉淀。

安装 / 上手步骤

  1. 下载 Obsidian，新建 Vault，建议放在固定目录。
  2. 建立 00-Inbox、10-Projects、20-Knowledge、90-Archive 四个目录。
  3. 安装 Dataview、Omnisearch 等插件，先保持少而稳。
  4. 每周把飞书、PDF 和聊天记录整理成主题卡片。
02 团队协作

飞书云文档与多维表格

用飞书承接团队协作、课程素材、会议纪要和客户项目，让资料能被搜索、追踪和复用。

适合场景适合课程共创、客户项目管理、社群运营和资料外发。

安装 / 上手步骤

  1. 安装飞书客户端，建立一个工作空间。
  2. 用云文档承载内容，用多维表格管理人、事、资料。
  3. 为每个项目设置权限，区分内部协作和对外分享。
  4. 重要结论同步回 Obsidian，保持私人知识库可沉淀。
03 AI 编程

AI 编程与网页制作

用 Cursor / VS Code / Codex 把内容、设计和部署连成一条小闭环，适合个人网站、课程页和小工具。

适合场景适合想自己做网页、自动化工具和结构化内容交付的人。

安装 / 上手步骤

  1. 安装 Git、Node.js LTS、Python 和 VS Code 或 Cursor。
  2. 把项目放到固定目录，用 Git 管理每次改动。
  3. 本地预览时运行 python -m http.server 8123，用浏览器检查。
  4. 修改完后执行 git add、git commit、git push，把网站同步到 GitHub。
04 Agent 工作流

AI Agent 任务拆解

把重复任务拆成目标、输入、工具、检查点和交付物，让 AI 不只回答问题，而是完成任务。

适合场景适合报告分析、资料整理、课程生成和项目执行。

安装 / 上手步骤

  1. 先写明白输入材料、最终交付物和判断标准。
  2. 把大任务分成搜索、阅读、提炼、写作、验证几步。
  3. 每一步都留下检查点，防止看似完成但不能交付。
  4. 先用小样本跑通，再扩展到批量任务。
05 资料提炼

报告阅读与案例提炼

把 PDF、网页、会议记录转成课程案例，每个结论都保留出处和可复核证据。

适合场景适合课程备课、客户研究、行业报告解读和内容写作。

安装 / 上手步骤

  1. 保留原始 PDF 或网页链接，先不急着改写。
  2. 提取目录、图表、关键结论和可引用数据。
  3. 把结论改写成课堂案例、提问和练习任务。
  4. 最后检查数据、时间和来源，确保可追溯。
06 自动化验收

浏览器自动化与网页检查

用 Playwright 检查页面、截图、抓取链接和复现用户操作，适合网站发布前的验收。

适合场景适合检查链接、响应式布局、PDF 访问和前端交互。

安装 / 上手步骤

  1. 先安装 Node.js LTS，确认 npx --version 可用。
  2. 用 npx --package @playwright/cli playwright-cli open <url> 打开页面。
  3. 用 snapshot 找到元素，再用 click、goto、eval 验证流程。
  4. 发布前检查控制台错误和移动端横向滚动。
07 网站发布

GitHub 与静态网站发布

把 HTML、PDF、课程页和报告发布到一个可长期访问的个人网站，保持可追溯、可回滚。

适合场景适合个人品牌、课程资料、项目作品和外发报告。

安装 / 上手步骤

  1. 在 GitHub 创建仓库，把本地网站推送到 master 或 main 分支。
  2. 在 GitHub Pages 或 Cloudflare Pages 绑定仓库。
  3. 配置 CNAME 和域名，确保 HTTPS 正常。
  4. 每次上线前先本地验收，再提交推送。
